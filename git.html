Guia Git e GitHub

O que é o Git

O Git é um sistema de controle de versão distribuído. Ele permite que desenvolvedores acompanhem mudanças no código-fonte durante o desenvolvimento de software. É extremamente eficiente para trabalhar em equipe.

O que é o GitHub

O GitHub é uma plataforma de hospedagem de código-fonte que utiliza o Git. Ele facilita a colaboração entre desenvolvedores, oferecendo ferramentas para revisão de código, controle de versão e gerenciamento de projetos.

Diferenças entre Git e GitHub

  • Git: ferramenta local de controle de versões.
  • GitHub: plataforma online que hospeda repositórios Git.
  • Você pode usar Git sem GitHub, mas não o contrário.

Principais comandos Linux para usar o Git Bash

  • ls: lista arquivos e diretórios.
  • cd: navega entre diretórios.
  • mkdir: cria um novo diretório.
  • rm: remove arquivos ou diretórios.
  • clear: limpa o terminal.
  • touch: cria um novo arquivo.
  • pwd: mostra o caminho do diretório atual.

Comandos Linux do repositório aula_git

  • mkdir aula_git
  • cd aula_git
  • git init
  • touch index.html
  • git status
  • git add .
  • git commit -m "meu primeiro commit"

Como criar um repositório no GitHub

  1. Acesse github.com e faça login.
  2. Clique em "New" para criar um novo repositório.
  3. Escolha um nome, visibilidade (público ou privado) e clique em "Create repository".

Como ligar em um projeto existente na máquina

  1. Abra o terminal na pasta do projeto.
  2. Inicie o repositório com git init.
  3. Adicione os arquivos com git add ..
  4. Faça o commit inicial com git commit -m "Primeiro commit".
  5. Conecte ao GitHub com git remote add origin <url_do_repositório>.

Como gerar uma versão com git

Para gerar uma versão (tag), utilize:

git tag -a v1.0 -m "Versão 1.0"

E envie para o GitHub com:

git push origin v1.0

Como enviar da máquina para o GitHub

  1. Verifique se o repositório está configurado: git remote -v
  2. Envie os commits com: git push origin main
  3. Se necessário, use git branch -M main para definir o nome da branch principal.
